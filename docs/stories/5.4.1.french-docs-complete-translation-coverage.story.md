# Story 5.4.1: French Documentation — Complete Translation Coverage

## Status

**Backlog** — Deferred from Story 5.4 (v1.1), targeted for v2.0.

---

## Story

**As a** French-speaking user browsing the documentation site,
**I want** all pages under `/fr/` to be in French (or clearly marked as English-only),
**so that** I don't encounter unexpected untranslated English content when navigating the French documentation.

---

## Acceptance Criteria

1. **AC1:** French CLI Reference available (`docs/CLI-REFERENCE.fr.md`) with translated prose, examples kept bilingual
2. **AC2:** French API Reference available (`docs/api-reference.fr.md`) with translated descriptions (code signatures remain English)
3. **AC3:** French Methodology page available (`docs/methodology.fr.md`) with translated explanations
4. **AC4:** All `/fr/` pages display content in French — no silent English fallback
5. **AC5:** MkDocs site builds without warnings for missing `.fr.md` files
6. **AC6:** Native French speaker review for translation quality (consistent with Story 5.4 quality bar)

---

## Context

Story 5.4 delivered French translations for 5 of 8 documentation pages (Home, Installation, Quick Start, FAQ, Troubleshooting). Three pages were explicitly deferred as out of scope:

| Page | File | Reason Deferred |
|------|------|-----------------|
| CLI Reference | `CLI-REFERENCE.md` (885 lines) | Complex, user-facing but developer-oriented |
| API Reference | `api-reference.md` | Developer-facing |
| Methodology | `methodology.md` (311 lines) | Academic/technical, English acceptable |

**The problem:** The `mkdocs-static-i18n` plugin with `docs_structure: suffix` falls back to the English source when no `.fr.md` file exists. This means `/fr/api-reference/`, `/fr/cli-reference/`, and `/fr/methodology/` all serve English content under French navigation labels — a confusing UX.

**Priority:** LOW — cosmetic/UX improvement, not a blocker for French-speaking users who can read technical English.

**Estimated effort:** ~3-4 days (translation + review)
