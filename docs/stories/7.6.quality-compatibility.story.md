### Python 3.13 Blockers (Expected)

Two dependencies currently block Python 3.13:
1. **PySide6-Essentials** — pinned to `python = ">=3.10,<3.13"` in `pyproject.toml` (line 43). Even if PySide6 6.8+ adds 3.13 support, our pin to `~6.7.0` would need updating [Source: pyproject.toml]
2. **spaCy/thinc** — historically blocked on newest Python versions due to compiled Cython extensions. Check `pypi.org/project/thinc` for 3.13 wheel availability [Source: architecture/3-tech-stack.md, project memory]

The `pyproject.toml` Python constraint is already `>=3.10,<3.14` so the overall project constraint allows 3.13; the blocker is individual dependency compatibility. [Source: pyproject.toml line 30]

- [x] `test_large_dataset_listing_non_blocking` non-blocking assertion is weak — `event_processed[0]` is set by a direct call to `track_events()` in the main thread, so it will always be True regardless of whether the worker blocks. Consider either removing the flag check or using a `QTimer.singleShot` callback to prove event loop responsiveness during worker execution.
