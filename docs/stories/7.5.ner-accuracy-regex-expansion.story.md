### Geography Disambiguation Design

**Current behavior:** `_match_geography()` in `regex_matcher.py` matches any capitalized token found in the geography dictionary. No POS-tag check.

**Target behavior (AC3):** Add POS-tag disambiguation:
1. Token must be capitalized (already enforced by regex pattern `[A-Z...]`)
2. If spaCy Doc is available: token's POS must be PROPN, OR token must not already be assigned an entity type by spaCy (i.e., `ent_type_ == ""`)
3. If spaCy Doc is not available: accept based on capitalization alone (backward compat for unit tests)

**Implementation approach:**
- Add optional `spacy_doc` parameter to `_match_geography()` and `match_entities()`
- In `_match_geography()`, for each candidate match, find the corresponding token(s) in the spaCy Doc by character offset
- Check POS tag and entity assignment
- Only emit the `DetectedEntity` if disambiguation passes
