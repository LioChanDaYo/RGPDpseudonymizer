# Story 7.1: Validation Workflow Productivity

## Status

**Done**

---

## Story

**As a** user validating entities in a document,
**I want** accepting or rejecting one occurrence of an entity to apply to all occurrences, and I want to hide already-confirmed entities,
**so that** I can complete validation faster, especially in documents with repeated names.

---

## Acceptance Criteria

1. **AC1 (FE-020):** Accepting one occurrence of an entity (e.g., "Marie Dupont") confirms ALL occurrences of the same entity text in the document
2. **AC2 (FE-020):** Rejecting one occurrence rejects all occurrences of the same entity text
3. **AC3 (FE-020):** Undo reverses the group action — all occurrences revert to previous state
4. **AC4 (FE-020):** Grouping uses exact text match only — different entity texts are NOT grouped (e.g., "Marie" and "Marie Dupont" remain independent). No fuzzy matching in v2.1.
5. **AC5 (FE-020):** Works from keyboard navigation (Enter/Delete) and context menu. Bulk selection (checkbox-based) operates on explicitly selected entities only — no implicit grouping.
6. **AC6 (FE-020):** Sidebar and document highlights update immediately for all affected occurrences
7. **AC7 (FE-019):** "Masquer les validees" checkbox visible next to existing "Masquer les rejetees" toggle
8. **AC8 (FE-019):** When toggled on, confirmed, added (user-created), and known entities are hidden from document highlights
9. **AC9 (FE-019):** Pending and rejected entities remain visible
10. **AC10:** No regression in existing bulk accept/reject by type or checkbox selection
11. **AC11:** Unit tests for grouping logic, undo/redo, and hide-confirmed toggle

### Implementation Pattern: Grouping Index

The grouping index should be built during `init_from_detection_result()` after the review loop:

```python
# Build text-based grouping index for validate-once-per-entity
self._text_group_index: dict[str, list[str]] = {}
for review in self._session._entity_reviews:
    text = review.entity.text
    if text not in self._text_group_index:
        self._text_group_index[text] = []
    self._text_group_index[text].append(review.entity_id)
```

Then in `accept_entity()`:

```python
def accept_entity(self, entity_id: str) -> None:
    review = self._reviews_by_id.get(entity_id)
    if review is None:
        return
    group_ids = self._text_group_index.get(review.entity.text, [entity_id])
    if len(group_ids) > 1:
        self._undo_stack.beginMacro("Accepter le groupe")
        for eid in group_ids:
            cmd = _SingleActionCommand(self, eid, EntityReviewState.CONFIRMED)
            self._undo_stack.push(cmd)
        self._undo_stack.endMacro()
    else:
        cmd = _SingleActionCommand(self, entity_id, EntityReviewState.CONFIRMED)
        self._undo_stack.push(cmd)
```

Same pattern for `reject_entity()`.

### Implementation Pattern: Hide-Confirmed Toggle

is_confirmed = review.state in (

### Important: Do NOT Modify Bulk Actions

`bulk_accept()`, `bulk_reject()`, `accept_all_of_type()`, and `accept_all_known()` operate on explicit entity_id lists provided by checkbox selection or type filtering. These must NOT gain implicit grouping behavior — the user explicitly chose those specific entities.

### Technical Constraints

- **PySide6 QUndoStack macros:** `beginMacro()`/`endMacro()` already used by `bulk_accept()`/`bulk_reject()`. Undo of a macro reverts ALL commands within it as a single step. This is exactly the behavior needed for AC3.
- **Signal emission during macros:** Each `_SingleActionCommand.redo()` emits `state_changed`. This means the panel/editor update per entity within the macro. For small groups this is fine; for 20+ entities, may need optimization (batch refresh after macro).
- **Existing variant occurrences:** `_variant_occurrences` maps canonical key → all `DetectedEntity` instances for document highlight expansion. This is orthogonal to text-based grouping — variants handle highlight positions, grouping handles validation actions.
