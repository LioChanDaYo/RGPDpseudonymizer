### Alpha Testing Protocol Template

**Test Scenario 1: Single Document**
1. Create a 5-page French document with 20-30 entities (names, locations, orgs)
2. Run: `poetry run gdpr-pseudo process document.txt`
3. Validate entities using UI
4. Check pseudonymized output quality
5. Rate experience (1-5): Installation, UI, Output Quality

**Test Scenario 2: Batch Processing**
1. Create 3-5 documents with overlapping entities (e.g., "Marie Dubois" appears in multiple docs)
2. Process each document sequentially
3. Verify "Marie Dubois" receives same pseudonym across all documents
4. Rate consistency and usability

**Test Scenario 3: Reversibility (Optional)**
1. Pseudonymize a document
2. Use mapping table to de-pseudonymize (if feature exists)
3. Verify original text restored

**Feedback Questions:**
1. Installation difficulty (1=easy, 5=very hard)
2. Validation UI usability (1=confusing, 5=intuitive)
3. Output readability (1=poor, 5=excellent)
4. Processing speed (acceptable/too slow)
5. Top 3 missing features
6. Would you use in production? (yes/no/maybe + why)
7. Open feedback (what worked well, what didn't)
