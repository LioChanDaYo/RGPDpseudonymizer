## 🎯 Release Scope

v2.0 introduces the **desktop GUI** with **WCAG 2.1 Level AA accessibility compliance**, complementing the existing CLI interface.

**Key Features:**
- Complete GUI workflow (document processing, entity validation, batch processing)
- Full accessibility support (keyboard navigation, screen reader, high contrast mode)
- Internationalization (French UI with English fallback)
- Configuration management and batch processing
- Visual entity validation interface

## 🚨 Known Limitations (Ship with v2.0)

### UI-001: 200% DPI Scaling Layout Issues

**Impact:** Low - Minor widget overlapping/squishing at 200% system DPI scaling
**Affected Users:** Users with extreme accessibility needs (200%+ scaling)
**WCAG Compliance:** PASS - Text remains readable, all functionality accessible
**Mitigation:** Consider layout refinements in v2.1 based on user feedback
**Documented:** [stories/6.7.accessibility-wcag-aa.story.md](stories/6.7.accessibility-wcag-aa.story.md#L469-L475)

### Version Bump
```bash
# Update version in pyproject.toml (single source of truth — no __version__ in __init__.py)
vim pyproject.toml

# Lock dependencies
poetry lock --no-update

# Verify version
poetry version
```

### PyPI Publication
```bash
# Build distribution
poetry build

# Optional: Test on test.pypi.org first
poetry publish -r testpypi

# Production publish
poetry publish

# Verify installation
pip install --upgrade gdpr-pseudonymizer
gdpr-pseudo --version  # Should show v2.0.0
```

### Testing Fresh Install
```bash
# Create clean virtualenv
python -m venv test_v2_install
source test_v2_install/bin/activate  # Windows: test_v2_install\Scripts\activate

# Install from PyPI
pip install gdpr-pseudonymizer==2.0.0

# Test CLI
gdpr-pseudo --help

# Test GUI
python -m gdpr_pseudonymizer.gui

# Deactivate
deactivate
```
